今天主要修复了什么

  • 长录音内存增长：录音过程改为实时写入临时 WAV，不再把全部音频帧一直堆在内存里；转写结束后自动清理临时文件。
  • 重复触发与重复请求：增加处理状态锁和 Windows 单实例保护，避免连续按热键时出现重复转写、重复 AI 请求或多个实例抢占快捷键。
  • 剪贴板稳定性：改用 Windows 原生接口读取文字；写入失败时停止自动粘贴，不再出现“实际失败但界面提示成功”。
  • 配置与密钥安全：API Key 使用 Windows DPAPI 加密；配置保存采用临时文件验证、原子替换和备份副本，降低配置损坏风险。
  • AI 接口容错：补上 URL 校验、连接和读取超时、限流、权限、空返回和内容截断处理；网络失败时保留本地整理结果。
  • 安装与打包：移除没有实际使用的大型依赖，构建过程使用独立环境，并检查外部命令的真实退出结果。

我的修复过程

1. 先确定边界

保留原来的界面、F2 热键和“本地识别 → 可选润色 → 自动粘贴”体验，只修稳定性、安全性和部署问题。

2. 从长期运行问题开始

先处理录音内存和临时文件，再加最长录音时间与处理锁，避免问题在连续使用中被放大。

3. 补齐失败路径

重点检查剪贴板、配置写入、AI 请求、安装脚本和打包脚本在失败时是否会给出真实结果。

4. 做安全收尾

加密 API Key、限制配置输入、保留配置备份，同时明确模型、个人配置和历史记录不进入分发包。

5. 用真实流程验证

不仅检查代码能否编译，还验证本地模型、真实录音、剪贴板、单实例、源码界面和最终 EXE。

02

Visual Style Study

提取“绛霞绮梦”：把感觉变成可重复使用的视觉语言

第二项工作是从三张参考图中提取一种东方古风柔雾摄影风格。我把模糊的“好看、梦幻、高级”拆成了模型能够理解的色彩、光线、材质、边缘、景深和构图规则。

这种风格能产生什么效果

它适合生成高完成度的写实东方时尚人像：人物皮肤明亮柔润，暖粉丝绸与半透明薄纱形成流动层次，刺绣、珍珠和香槟金饰品只在局部闪光；背景压入深棕和棕紫，再用窗光、灯笼暖光、浅景深、轻雾和前景纱幔制造朦胧空间。

低饱和暖粉 裸杏 浅胭脂 藕荷紫 香槟金 棕紫暗部

以上为本次风格提取使用的参考素材，用于展示目标视觉效果，不代表本次新生成作品。

这次真正沉淀下来的成果

  • 一份中文通用提示词和一份英文通用提示词。
  • 固定的基础风格层、光线色彩层、构图层和负面约束层。
  • 可替换的人物、动作、服装与场景内容区，方便继续生成系列图。
  • 对当前证据范围的说明：人像与丝纱材质已经有参考支持，但无人环境、工业产品和高速动作仍需要继续测试。

这条视频让我确认的事情

  • AI 视频不只是“生成几个镜头”，真正决定完成度的是脚本、资产一致性、声音、节奏和反复质检。
  • 剪纸视觉非常适合历史、城市文化和知识科普，它能避开写实人物不稳定的问题，同时形成明确的系列识别度。
  • 把制作流程写成代码后，同类主题可以复用结构，下一次主要替换故事、图片资产和旁白内容。
今天最大的收获： 三件事情看起来完全不同——桌面工具、视觉风格、AI 视频——但它们背后其实是同一种工作方法：先把模糊需求拆清楚，再把过程做成可验证、可复用、可持续迭代的系统。

从零搭建个人博客：运营人的技术实践

技术 博客 DIY

你现在看到的这个博客，是我——一个完全不懂编程的运营人——从零搭建起来的。如果你也是非技术背景但想拥有一个属于自己的网站，这篇文章就是为你写的。我想证明一件事：在 AI 时代，不会写代码已经不再是拥有个人网站的障碍了。

一、为什么我要自己搭博客

2025 年初，我在几个平台上零散地发了一些文章，虽然也有读者反馈，但总觉得受制于人——平台改规则、限流、甚至莫名其妙删文，这些事都让我很不爽。更重要的是，我不想把自己的内容资产全放在别人的平台上。我希望有一个完全属于自己的空间，想怎么写就怎么写，想怎么设计就怎么设计。

有了这个念头之后，我去了解了一下搭建博客的方案，发现主要有三条路：

  • WordPress：功能最全但太复杂，需要服务器和数据库，还要处理各种插件兼容问题
  • 静态博客生成器：Hexo、Hugo、Jekyll 这些，需要懂命令行操作，对我这种小白还是有点门槛
  • 纯手工 HTML：一个字一个字地写 HTML 文件，上传到服务器就行。看起来最原始，但反而是最可控的

我最后选了第三条路。听起来很夸张对吧？但在 AI 的帮助下，这条路其实比想象中简单得多。

二、我用 AI 学会了写 HTML

一开始我对 HTML 的认知仅限于"它是网页的源代码"。我没学过任何编程语言，连 HTML 标签都不认识几个。但我发现了一个规律：AI 是最好的编程老师，因为它不会嫌弃我的问题太蠢。

我的学习过程是这样的：

  • 第一步：我问 ChatGPT "HTML 网页的基本结构是什么？"它给我了一个最简单的例子，我复制到本地打开，看到了一个网页诞生了。
  • 第二步：我说"我想把标题变成蓝色"，它告诉我去掉 style 属性，加什么 class，引入 Tailwind。
  • 第三步：我说"我想在右上角加一个导航栏"，它给我了一段代码，我粘贴上去改了改就生效了。
  • 第四步：每遇到一个问题，我就直接问 AI。不知不觉中，我学会了 div、class、flex、grid 这些概念。

不到两周，我就能从零写一个完整的网页了。不是说我现在成了程序员，而是我学会了"用 AI 来写代码"这个元技能。这个能力比学会某个具体的语法重要得多。

💡 给非技术朋友的建议

不要一开始就想系统学习编程。而是先想清楚你要做什么，然后让 AI 帮你实现，过程中自然就学会了。需求驱动学习是最快的路径。你需要知道的不是"怎么写代码"，而是"怎么让 AI 帮你写代码"。

三、我的技术方案

如果你也想搭一个类似的博客，这里是我的全套技术方案：

  • 托管：Vercel（免费版），连接 GitHub 仓库后自动部署
  • 域名：在 Namesilo 买的，一年几十块
  • 样式：Tailwind CSS（通过 CDN 引入，不需要安装任何东西）
  • 字体图标：Font Awesome（同样是 CDN 引入）
  • 文章编辑：直接写 HTML 文件，用 AI 辅助生成内容结构和样版
  • 版本管理：GitHub，免费且稳定

整套方案零成本（域名除外），不需要服务器、不需要数据库、不需要装任何软件，一个记事本加上浏览器就够了。

四、搭建过程的踩坑记录

过程当然不是一帆风顺的。分享几个我踩过的坑：

  • 坑一：第一次上传文件到 Vercel，发现样式全乱了。折腾半天发现是路径问题——在本地好好的，上传到服务器后 CSS 和图片的引用路径就不对了。我的解决方案是用相对路径代替绝对路径。
  • 坑二：Google Fonts 加载太慢。我引入了一个中文字体，结果页面加载要好几秒。最终决定保留西文字体用 Google Fonts，中文字体用系统默认，保证加载速度。
  • 坑三：移动端适配。第一次做出来的网页在电脑上看着还行，手机上完全没法看。后来学会了加 viewport meta 标签和响应式 class，才解决了这个问题。

每个坑对应我学到的一个新知识。从这个角度看，踩坑不是坏事，反而是最好的学习机会。

五、我想对你说的话

写这篇文章的初衷，是想鼓励和我一样的非技术背景的朋友：不要被"编程"这两个字吓到。搭一个个人博客所需的技术知识，远没有你想象的那么多。在 AI 的帮助下，一个零基础的人两周就能上线自己的网站。

拥有一个自己的网站有什么好处？对我的改变是：

  • 我开始更系统地整理自己的知识和经验（因为要写出来给别人看）
  • 我建立了一个属于自己的"数字资产"，不再依赖任何第三方平台
  • 我收到了一些意外的合作邀请和面试机会
  • 我发现自己比想象中更有动手能力，这种信心也延伸到了其他领域
技术从来没有像今天这样平易近人。如果你想做一件事，现在的 AI 几乎可以帮你克服任何技术障碍。唯一的限制只有你想不想做。

如果你看了这篇文章也想去试试，我的建议很简单：今天就注册一个 GitHub 账号，然后把你想写的文章用最简单的 HTML 写出来。迈出第一步，你已经超过了 99% 只停留在"想想"阶段的人。
